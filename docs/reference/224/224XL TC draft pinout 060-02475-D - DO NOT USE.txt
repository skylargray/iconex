---PLAN---
the timing work already lets us skip the single densest part of T&C.
Here's the organizing principle, then a ranked trace list and an explicit ignore list.

The principle that shrinks the job
The model needs two kinds of T&C signal:

-Pure-timing signals (MC, MS0–8, AS0/1/2, ARUCK, ARUCKE/, DAB WSTB, DAB RSTB, S0/S1).
Their waveforms are already pinned in 224XL_timing_spec.md (figs 3.2/3.4), and their consumers
are already mapped in the ARU/DMEM netlist. The model generates these from a counter.
You do not need to trace how they're generated.
-Microword-decoded signals (the offset, the coefficient, RA/WA, CSIGN, XFER/ZERO enables, the device-decode)
 — these carry the program's meaning and are not derivable from any timing diagram.
 This is what's worth tracing.
So: trace the decode and the WCS-bit→destination wiring; skip the clock/PLL generator.

Trace priority (highest leverage first)

P0 — the device-decode cluster (small, and it defines what every step does)
-tc_U47 (LS139) + tc_U48 (LS00) + tc_U49 (LS10, incl. U49C = WR DA/) + tc_U34A.
-Decodes (MI16,MI17,MI12,MI13,MI4,MI7) → MEMAC, MEMR, MEMW/, RDRREG/, RD AD/, WR DA/ and the DAB-driver enables.
-Why first: it's a handful of chips, and it tells you per step whether it's a DRAM read/write / coeff / I/O,
and who drives the DAB — which closes net-group 1 and resolves the dead-tank "which steps write DMEM" question.
It also lets us confirm or correct the believed decode
(the MEMAC=MI17 / sub-decoder beliefs that are currently ❌-FAITH in the bottom-up work).

P0 — the offset path (this is the delay)
-The WCS offset-field latch → OFST0–15/ → out to the DMEM adder B (dmem_U49/50/63/64, §5.2).
-Why: offset = delay length. Highest reverb-critical net there is.

P0 — the coefficient serializer (this is the gain / the multiplier)
-The WCS coefficient bits → the 74195/74194 shifters → M0/ M1/ (ARU Booth, §4F.5/4F.6) across the 3 AS states,
plus CSIGN/.
-Why: coeff = gain; M0/M1 is how it enters the modified-Booth multiplier. Pinning this resolves the
coeff/serialization uncertainty in the field map.

P1 — the regfile addresses
-WCS bits → RA0/RA1/WA0/WA1 (ARU regfile read/write select, §4F.1/4F.6) = which delay-line slot each step
touches.

P1 — the decode-gated strobes
-Which microword bit gates the fixed clock to produce XFER CK and ZERO/ (accumulator transfer / clear),
and the MEMAC gating of CAS. The timing is known; you only need the enable term (which MI bit ANDs with
which edge). Much of this falls out of P0's decode.

P2 — clock fan-out sanity (light, not a real trace)
-Just confirm the net names ARUCK / ARUCKE/ / DAB WSTB / DAB RSTB / S0 / S1 come off the expected generator
outputs, cross-checked against fig-3.2/3.4. No gate-level work — a labeling pass.

Can be ignored (for the reverb model)
-The entire clock/PLL generator gate-level — mc4044 (U27 PLL), the ÷2 JK (U26), the ÷9 counter (U56),
the MS shift register (U39), the AS-grouping (U38/U25/U23). Output = the timing spec; the model makes
MS/AS from a counter. (Only a cycle-accurate boot/PLL-lock emulation would need it — not reverb.)
-FPC clock/control generation on T&C — FPC is scoped out (we only need its number-format + DAB in/out
step later).
-SBC bus handshake / wait-states / arbitration beyond the P0 device decode — that's the program-load path,
and we already load programs via the firmware boot (boot8080). Not needed for per-sample DSP.
-WCS storage array (mcm68b10 + am8304) gate-level — the microword contents come from the firmware image;
you need the output-bit→destination map (P0/P1), not the RAM wiring.
-Test / diagnostic / POST-only logic.

Suggested order for you to hand-trace
1. Device decode (tc_U47/48/49/34A) — small, unlocks the most.
2. Offset latch → OFST/.
3. Coeff serializer → M0//M1/ + CSIGN/.
4. RA/WA regfile addresses.
5.XFER/ZERO enable terms.
6. (light) clock-name fan-out confirm.

Net effect: the dense clock-generation machinery — the part that looks most intimidating — is exactly the
part you can skip. What's left to trace is a decode cluster + a handful of field→destination paths,
and tracing them would also definitively pin the microword field map that the bottom-up reverb work is
currently taking on faith.
